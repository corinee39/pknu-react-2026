# SPA (Single Page Application) 개요

## 1. SPA 무엇인가?

**SPA (Single Page Application, 단일 페이지 애플리케이션)** 는 사용자 인터페이스에서 페이지를 새로고침하지 않고도 다양한 콘텐츠를 제공하는 웹 애플리케이션 형태입니다.

### 핵심 특징
- **페이지 새로고침 불필요**: 브라우저가 URL 을 변경하고 서버로 요청할 때, 기존 DOM 을 유지하면서 콘텐츠를 즉시 렌더링합니다.
- **즉각적인 응답**: 서버와 클라이언트를 직접 연결하여 데이터를 교환하고, 변경된 DOM 을 렌더링합니다.
- **앱처럼 동작**: 모바일 앱처럼 사용자 경험을 제공합니다.

### 전통적 웹과 비교

| 비교 항목 | 전통적 웹 (MPA) | SPA |
|-----------|-----------------|-----|
| 페이지 전환 | 전체 페이지 재로드 | 콘텐츠 교체만 |
| 서버 요청 빈도 | 각 페이지마다 | 필요시만 |
| 페이지 전환 속도 | 느림 (서버 요청 + 렌더링) | 빠름 (클라이언트 렌더링) |
| 초기 로딩 속도 | 빠름 | 느림 (코드 bundles 로딩) |

## 2. React 의 장점

### 2.1 컴포넌트 기반 구조
- **재사용성**: 컴포넌트는 독립적인 UI 구성 요소로 재사용 가능합니다.
- **코드 조직**: 큰 애플리케이션을 작은 컴포넌트 단위로 나누어 관리하기 쉽습니다.
- **테스트 용이**: 컴포넌트는 단위 테스트에 적합합니다.

```jsx
// 컴포넌트 재사용 예시
const Button = ({ label, onClick }) => {
  return <button onClick={onClick}>{label}</button>;
};

// 여러 컴포넌트를 재사용
<Button label="확인" onClick={handleConfirm} />
<Button label="취소" onClick={handleCancel} />
```

### 2.2 가상 DOM (Virtual DOM)
- **성능 최적화**: 실제 DOM 을 변경하기 전에 가상 DOM 을 먼저 업데이트하고, 차이점을 계산하여 효율적으로 렌더링합니다.
- **배타성**: 불필요한 DOM 조작을 최소화하여 성능을 향상시킵니다.

### 2.3 강력한 개발 도구 생태계
- **React DevTools**: 컴포넌트 구조, 상태, props 를 시각적으로 확인할 수 있습니다.
- **자바스크립트 기반**: 타입 체크, 린팅, 코드 조직 도구와의 호환성이 뛰어납니다.
- **커뮤니티 지원**: 방대한 양의 튜토리얼, 라이브러리, 플러그인이 제공됩니다.

### 2.4 데이터 흐름의 명확성
- **React State**: 컴포넌트 상태가 명확하게 정의되어 유지보수가 용이합니다.
- **One-way 데이터 흐름**: 데이터를 상위 컴포넌트에서 하위로 흐르게 하여 예측 가능한 구조를 제공합니다.

```jsx
// 일방향 데이터 흐름 예시
function UserList({ users }) {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### 2.5 React Router 와 통합
- **클라이언트 측 라우팅**: SPA 에서 페이지 전환 시 라우팅을 관리할 수 있습니다.
- **URL 과 상태 연동**: URL 을 변경할 때 상태도 함께 관리할 수 있습니다.

### 2.6 생태계 라이브러리와의 호환성
- **Material-UI, Ant Design**: UI 컴포넌트 라이브러리와의 호환성이 뛰어납니다.
- **상태 관리 도구**: Redux, Context API, Zustand 등 다양한 상태 관리 솔루션과 호환됩니다.

## 3. SPA vs MPA (Multi-Page Application)

### SPA 특징
- 장점: 빠른 페이지 전환, 부드러운 사용자 경험
- 단점: 초기 로딩 속도 느림, 서버 부담 증가
- 사용 사례: 복잡한 대시보드, 애플리케이션 위주의 사이트

### MPA 특징
- 장점: 빠른 초기 로딩, 서버 부담 감소
- 단점: 페이지 전환 시 끊김 느낌, 전체 페이지 재로드 필요
- 사용 사례: 정보 제공 위주의 웹사이트, 블로그

## 4. SPA 의 발전

| 시대 | 기술 | 특징 |
|------|------|------|
| 2010 년대 이전 | Flash, AJAX | Flash 기반 인터랙티브 콘텐츠 |
| 2010 년대 | AngularJS | 서버 렌더링 위주 |
| 2013 년대 | React, Angular, Vue | 클라이언트 측 렌더링 등장 |
| 2015 년대 | React Router | 클라이언트 측 라우팅 표준화 |
| 2020 년대 | Next.js, Nuxt.js | 서버 렌더링 (SSR) 도입 |

---

**참고**: SPA 는 웹 애플리케이션의 미래를 열고 있습니다. 초기 로딩 문제는 서버 렌더링 (SSR) 및 스테이트스레드 (SSG) 기법으로 해결되었습니다.
